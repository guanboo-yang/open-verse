# 綱目 (outline) 已知問題記錄

來源網站 recoveryversion.com.tw 的綱目原始資料有一些不一致或疑似錯誤之處。
以下記錄供日後逐步修正（scrape_outline.py 產生 outline.json）。

## 1. marker 跑到標題格（site 資料錯位）

marker 應該在獨立的 `<TD title='N'>` 格，但少數情況 marker 文字跑進了標題格，
導致 `marker` 欄位為空、編號殘留在 `title` 裡。

- `05_023`（申命記 23）L3：`21 關於鄰舍的出產　二三24∼25`
  - 現況：`marker=""`、`title="21 關於鄰舍的出產"`、`range="二三24∼25"`
  - 應為：`marker="21"`、`title="關於鄰舍的出產"`
- `04_033`（民數記 33）L2：`十　（豫先安排美地的分配－續）`
  - marker「十」外洩到標題格；目前 parse_continued 取出 `marker="十"`，
    但 title 仍保留外層括號 `（豫先安排美地的分配－續）`。

## 2. 「續」延續重述標記

某段綱目跨多章時，網站在新章頁頂重述父段並加「－續」。例如創 25∼28 反覆出現
「（以撒的經歷－續）」。這些**已抓進** outline.json 並標記 `continued: true`
（46 筆），UI 以較淡顏色顯示。

注意：少數續標記**沒有**內層 marker（如「（以撒的經歷－續）」marker 為空），
這是原始資料就沒有，非解析問題。

## 3. 括號補充段落（無「續」）

恢復本用括號標示補充性段落（家譜等），有真實範圍、非延續。已正常抓取。

- `01_036`（創 36）L4：`（以掃的後代　三六1∼43）` → marker 空、range `三六1∼43`
  - 這類 marker 為空是原始資料就無，待確認是否該補。

## 4. 顯示用 range 的非標準寫法（不影響錨點）

錨點用「文件順序」定位，以下只影響 `range` 顯示字串，皆原樣保存：

- 跨冊範圍含書名前綴：`撒上十六1∼撒下二四25`（撒上/撒下、王上/王下、代上/代下）
- 章節間有半形空格：`二四 12∼18`
- 逗號多節：`二二5，9∼12`、`二十1∼13，23∼29，二一4∼9`
- `上/下` 切段：`十一43上`、`一2下`
- 章號用 positional 中文：`二四`=24、`二五`=25（非二十四）

## 5. 無範圍標題

啟示錄等有 7 個純標題、無範圍（如「七個金燈臺─發光的召會」），`range` 留空，
屬正常。詩篇 5 個卷標題用 `</BR>` 分隔 `第X至Y篇` 與描述（已特別處理）。
